## 1. Project Overview

This project simulates a retail company selling products across multiple stores and provides analytics on both historical and real-time sales activity.

The pipeline answers business questions such as:
- What is the total revenue?
- Which products generate the most revenue?
- Which cities and stores perform best?
- What is the average basket value?
- How are real-time sales evolving?

---

## 2. Business Use Case

Retail companies need to collect, clean, transform and analyze sales data from different sources. In this project, sales data is generated in two ways:

1. **Batch data** — historical sales generated as CSV files
2. **Streaming data** — real-time sales events generated by a Kafka producer

The final goal is to provide business-ready indicators through analytical tables and a live dashboard.

---

## 3. Architecture

### Batch Pipeline

```text
CSV Data
   ↓
Python ETL scripts
   ↓
PostgreSQL (raw layer)
   ↓
SQL transformations
   ↓
PostgreSQL (staging layer)
   ↓
SQL analytics tables
   ↓
Streamlit dashboard
```

**Airflow DAG:** `sales_batch_etl_pipeline`

```text
generate_batch_data
        ↓
load_batch_to_postgres
        ↓
transform_raw_to_staging
        ↓
build_analytics
```

### Streaming Pipeline

```text
Python Kafka Producer
        ↓
Redpanda / Kafka topic: sales_stream
        ↓
Python Kafka Consumer
        ↓
PostgreSQL raw.sales_stream
        ↓
analytics.realtime_sales_metrics
        ↓
Streamlit dashboard
```

## 6. Database Design

The PostgreSQL database is organized into three layers.

### Raw Layer
Stores data close to its original format.

| Table |
|---|
| `raw.products` |
| `raw.stores` |
| `raw.sales_batch` |
| `raw.sales_stream` |

### Staging Layer
Contains cleaned and enriched data. The `staging.sales_clean` table joins sales with products and stores to add: product name, category, store name, city, region, sale date and sale hour.

### Analytics Layer
Contains business-ready tables consumed by the dashboard.

| Table |
|---|
| `analytics.daily_sales_summary` |
| `analytics.product_performance` |
| `analytics.realtime_sales_metrics` |

---

## 7. Batch ETL Pipeline

```text
generate_batch_data.py
        ↓
load_batch_to_postgres.py
        ↓
02_staging_to_clean.sql
        ↓
03_build_analytics.sql
```

The loading script is **idempotent** — it uses `ON CONFLICT DO UPDATE`, so the pipeline can be re-run safely without creating duplicates.

Run the full batch pipeline:

```bash
python scripts/run_batch_pipeline.py
```

---

## 8. ELT SQL Transformations

### Raw → Staging (`sql/02_staging_to_clean.sql`)

Joins `raw.sales_batch`, `raw.products` and `raw.stores` to produce `staging.sales_clean`.

### Staging → Analytics (`sql/03_build_analytics.sql`)

Builds `analytics.daily_sales_summary` and `analytics.product_performance`, computing:
- Total revenue
- Total orders and items sold
- Average basket
- Product performance
- Revenue by store and city

---

## 9. Airflow Orchestration

**DAG file:** `dags/sales_batch_pipeline_dag.py`  
**DAG name:** `sales_batch_etl_pipeline`

Configuration:
- Daily schedule
- 2 retries per task
- Clear task dependency order

Access the Airflow UI at **http://localhost:8080**

## 10. Streaming Pipeline

Uses **Redpanda** as a Kafka-compatible broker on topic `sales_stream`.

### Producer (`scripts/kafka_producer.py`)

Generates real-time sales events containing: sale ID, product ID, store ID, customer ID, quantity, unit price, total amount, payment method and timestamp.

### Consumer (`scripts/kafka_consumer.py`)

Listens to the topic, inserts events into `raw.sales_stream` and updates `analytics.realtime_sales_metrics` with:
- Recent revenue
- Recent number of sales
- Recent average basket

## 12. How to Run

### Quick Start

```bash
# 1. Start all services
docker compose up -d

# 2. Run the batch pipeline
python scripts/run_batch_pipeline.py

# 3. Launch the dashboard
streamlit run dashboard/app.py
```

### Streaming

```bash
# Terminal 1 — start the consumer
python scripts/kafka_consumer.py

# Terminal 2 — start the producer
python scripts/kafka_producer.py
```

> Full setup instructions are available in `SETUP.md`.
