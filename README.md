## 1. Project Overview

This project simulates a retail company selling products across multiple stores and provides analytics on both historical and real-time sales activity.

The pipeline answers business questions such as:

- What is the total revenue?
- Which products generate the most revenue?
- Which cities and stores perform best?
- What is the average basket value?
- How are real-time sales evolving?

The project includes both:

- a **batch ETL / ELT pipeline** for historical sales data
- a **real-time streaming pipeline** for live sales events

---

## 2. Business Use Case

Retail companies need to collect, clean, transform and analyze sales data from different sources.

In this project, sales data is generated in two ways:

1. **Batch data** — historical sales generated as CSV files
2. **Streaming data** — real-time sales events generated by a Kafka producer

The final goal is to provide business-ready indicators through analytical tables and a live dashboard.

## 6. Database Design

The PostgreSQL database is organized into three layers:

```text
raw       → raw data
staging   → cleaned and enriched data
analytics → business-ready tables
```

### Raw Layer

The raw layer stores data close to its original format.

| Table | Description |
|---|---|
| `raw.products` | Product reference data |
| `raw.stores` | Store reference data |
| `raw.sales_batch` | Historical batch sales |
| `raw.sales_stream` | Real-time streaming sales |

### Staging Layer

The staging layer contains cleaned and enriched data.

| Table | Description |
|---|---|
| `staging.sales_clean` | Clean sales enriched with product and store information |

The `staging.sales_clean` table joins sales with products and stores to add:

- product name
- category
- store name
- city
- region
- sale date
- sale hour

### Analytics Layer

The analytics layer contains business-ready tables consumed by the dashboard.

| Table | Description |
|---|---|
| `analytics.daily_sales_summary` | Revenue, orders and basket metrics by day and store |
| `analytics.product_performance` | Product revenue, quantity sold and order count |
| `analytics.realtime_sales_metrics` | Real-time sales metrics from Kafka events |

## 7. Batch ETL Pipeline

The batch pipeline generates synthetic retail data, loads it into PostgreSQL, then transforms it into analytics tables.

```text
generate_batch_data.py
        ↓
load_batch_to_postgres.py
        ↓
02_staging_to_clean.sql
        ↓
03_build_analytics.sql
```

Generated CSV files:

- `products.csv`
- `stores.csv`
- `sales_batch.csv`

The loading script is **idempotent**. It uses `ON CONFLICT DO UPDATE`, so the pipeline can be re-run safely without creating duplicate records.

Run the full batch pipeline:

```bash
python scripts/run_batch_pipeline.py
```

## 8. ELT SQL Transformations

The project also follows an ELT approach: data is first loaded into PostgreSQL, then transformed with SQL.

### Raw → Staging

File:

```text
sql/02_staging_to_clean.sql
```

This script joins:

- `raw.sales_batch`
- `raw.products`
- `raw.stores`

Output:

```text
staging.sales_clean
```

### Staging → Analytics

File:

```text
sql/03_build_analytics.sql
```

This script builds:

- `analytics.daily_sales_summary`
- `analytics.product_performance`

It computes:

- total revenue
- total orders
- total items sold
- average basket
- product performance
- revenue by store and city

## 10. Streaming Pipeline

The streaming pipeline simulates real-time sales activity.

It uses **Redpanda** as a Kafka-compatible broker.

Topic:

```text
sales_stream
```

### Producer

File:

```text
scripts/kafka_producer.py
```

The producer generates real-time sales events and sends them to the Kafka topic.

Each event contains:

- sale ID
- product ID
- store ID
- customer ID
- quantity
- unit price
- total amount
- payment method
- sale timestamp

### Consumer

File:

```text
scripts/kafka_consumer.py
```

The consumer listens to the Kafka topic, reads sales events and inserts them into:

```text
raw.sales_stream
```

It also updates:

```text
analytics.realtime_sales_metrics
```

The real-time metrics include:

- recent revenue
- recent number of sales
- recent average basket

## 12. How to Run

Full setup instructions are available in:

```text
SETUP.md
```

### Quick Start

```bash
# 1. Start all services
docker compose up -d

# 2. Run the batch pipeline
python scripts/run_batch_pipeline.py

# 3. Launch the dashboard
streamlit run dashboard/app.py
```

### Streaming

The streaming pipeline requires two terminals.

Terminal 1 — start the consumer:

```bash
python scripts/kafka_consumer.py
```

Terminal 2 — start the producer:

```bash
python scripts/kafka_producer.py
```

### Useful URLs

```text

Streamlit:    http://localhost:8501
PostgreSQL:   localhost:5433
```
